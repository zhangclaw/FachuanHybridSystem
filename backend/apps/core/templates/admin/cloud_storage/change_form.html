📖 WebDAV 配置指南 — 坚果云

第一步：获取坚果云应用密码

  1. 登录 坚果云官网
  2. 点击右上角用户名 → 账户信息
  3. 左侧菜单选择 安全选项
  4. 找到「第三方应用管理」区域，点击 添加应用密码
  5. 输入应用名称（如「法穿AI」），点击 生成密码
  6. 复制生成的密码（16位字母），粘贴到下方「WebDAV 应用密码」字段

第二步：填写本页配置

WebDAV 地址 https://dav.jianguoyun.com/dav/（默认已填）
WebDAV 用户名 你的坚果云注册邮箱
WebDAV 应用密码 第一步生成的16位密码（非登录密码）
WebDAV 根路径 默认 / 即可（指向坚果云 WebDAV 根目录）
如果只想绑定某个子文件夹，可填如 我的坚果云/工作文件
⚠️ 注意：坚果云免费版限制 600 次请求/30分钟，系统已内置限流（约3.5秒/次），正常使用不会超限。

📖 WebDAV 配置指南 — 123云盘

第一步：获取123云盘应用密码

  1. 登录 123云盘官网
  2. 进入 个人中心 → 安全设置
  3. 找到「应用密码」区域，点击 添加应用密码

  5. 复制生成的密码，粘贴到下方「WebDAV 应用密码」字段

第二步：填写本页配置

WebDAV 地址 https://webdav.123pan.com/dav/
WebDAV 用户名 你的123云盘账号（手机号或邮箱）
WebDAV 应用密码 第一步生成的应用密码（非登录密码）
WebDAV 根路径 默认 / 即可（指向123云盘 WebDAV 根目录）
如果只想绑定某个子文件夹，可填如 工作文件
💡 提示：123云盘 WebDAV 支持标准协议，免费版提供 2TB 存储空间。系统已内置限流保护，正常使用不会触发限制。
{# ── OneDrive 配置指南 ── #}
📖 OneDrive 配置指南

第一步：注册 Azure AD 应用

  1. 打开 Microsoft Entra 管理中心（原 Azure AD）
  2. 左侧菜单 → 标识 → 应用程序 → 应用注册
  3. 点击 新注册
  4. 名称填「法穿AI」（或任意名称）
  5. 支持的帐户类型选 「任何组织目录中的帐户和个人 Microsoft 帐户」
  6. 重定向 URI 留空（本系统使用设备码流，不需要回调地址）
  7. 点击 注册
  8. 在应用概览页，复制 「应用程序(客户端) ID」，填入下方「Client ID」

第二步：配置应用权限

  1. 在应用页面，左侧 → API 权限
  2. 点击 添加权限 → Microsoft Graph → 委托的权限
  3. 搜索并添加以下权限：
    • Files.ReadWrite — 读写用户文件
    • offline_access — 保持登录态
  4. 无需「授予管理员同意」，个人账号自己授权即可

第三步：填写本页配置并授权

Client ID 第一步复制的「应用程序(客户端) ID」
Tenant ID 默认 consumers（个人账号）
如果是企业账号，填你的租户 ID（如 xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx）
OneDrive 根路径 默认 /（OneDrive 根目录）
如只想绑定某个文件夹，填如 /工作/合同

第四步：完成授权

  1. 保存本页配置后，页面底部会出现「OneDrive 授权」区域
  2. 点击 获取设备码，页面会显示一个设备码和验证地址
  3. 在浏览器打开验证地址，输入设备码，用微软账号登录并同意授权
  4. 回到本页刷新，状态应变为 已授权
💡 提示：授权有效期约1小时，过期后系统会自动用 refresh_token 续期，无需手动操作。如果 refresh_token 也过期（通常90天），需要重新授权。

{# ── Google Drive 配置指南 ── #}
📖 Google Drive 配置指南

第一步：创建 Google Cloud 项目并启用 Drive API

  1. 打开 Google Cloud Console
  2. 创建新项目（或选择已有项目）
  3. 左侧菜单 → API 和服务 → 库，搜索并启用 Google Drive API

第二步：创建 Service Account

  1. 左侧菜单 → API 和服务 → 凭据
  2. 点击 创建凭据 → 服务账号
  3. 填写名称（如「法穿AI」），点击 创建并继续
  4. 权限步骤可跳过，点击 完成
  5. 在服务账号列表中，点击刚创建的账号 → 密钥 选项卡
  6. 点击 添加密钥 → 创建新密钥 → 选择 JSON → 创建
  7. 下载的 JSON 文件就是所需的凭证

第三步：共享文件夹给 Service Account

  1. 在 Google Drive 中，右键点击要使用的文件夹 → 共享
  2. 输入 Service Account 的邮箱地址（格式为 xxx@项目ID.iam.gserviceaccount.com，在凭据页面可见）
  3. 权限选 查看者 或 编辑者

第四步：填写本页配置

Service Account JSON 将下载的 JSON 文件内容完整粘贴到此字段（加密存储）
根文件夹 ID 从共享文件夹的 URL 中获取，格式如 https://drive.google.com/drive/folders/文件夹ID
默认 root 表示 Service Account 的根目录
根路径 默认 /
{# ── Dropbox 配置指南 ── #}
📖 Dropbox 配置指南

第一步：创建 Dropbox App

  1. 打开 Dropbox 开发者控制台
  2. 点击 创建应用
  3. 选择 Scoped access → Full Dropbox（或 App folder）→ 填写应用名称
  4. 在应用设置页面，找到 App key 和 App secret，复制到下方
  5. 切换到 权限 选项卡，勾选 files.content.write 和 files.content.read
  6. 点击 提交 以发布权限变更

第二步：填写本页配置并授权

App Key 第一步获取的 App Key
App Secret 第一步获取的 App Secret（加密存储）
根路径 默认 /（Dropbox 根目录）
如只想绑定某个子文件夹，填如 /工作/合同

第三步：完成授权

  1. 保存本页配置后，页面底部会出现「Dropbox 授权」区域
  2. 点击 获取设备码，页面会显示一个设备码和验证地址
  3. 在浏览器打开验证地址，输入设备码，用 Dropbox 账号登录并同意授权
  4. 回到本页刷新，状态应变为 已授权
💡 提示：Dropbox Access Token 有效期约4小时，过期后系统会自动用 Refresh Token 续期，无需手动操作。
