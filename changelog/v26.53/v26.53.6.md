# v26.53.6 (2026-06-23)

## 修复

### OA 立案 Playwright asyncio loop 崩溃

**问题**：OA 立案执行时，Playwright Sync API 在 Django ASGI 的 asyncio 事件循环内调用，抛出 `Error: It looks like you are using Playwright Sync API inside the asyncio loop`，导致立案功能完全不可用。

**根因**：`ScriptExecutorService` 在 `ThreadPoolExecutor` 工作线程中使用 `nest_asyncio.apply()` 修补事件循环，但该 monkey-patch 未能正确覆盖 Playwright 内部创建的新事件循环。

**修复**：将 OA 立案模块（`jtn/filing/`）全部从 Playwright Sync API 改为 Async API，彻底消除 sync/async 冲突。（`f0baf8b`、`82b543f`、`ab4ed79`）

## 重构

### 统一 JTN OA 登录服务 `JtnAuthService`

金诚同达 OA 原有三套独立的登录实现（立案、案件导入、客户导入），核心逻辑高度重复但各自不完整。本次重构抽取 `jtn/auth/` 公共模块，三个子模块统一调用。（`735e178`、`611027c`）

**新增文件：**

| 文件 | 职责 |
|------|------|
| `jtn/auth/__init__.py` | 导出 `JtnAuthService` |
| `jtn/auth/constants.py` | 公共常量（`_LOGIN_URL`、`_HTTP_HEADERS`、`_DEFAULT_HTTP_TIMEOUT`、`_COOKIE_PATH`） |
| `jtn/auth/service.py` | 统一登录服务（SSO 扫码、HTTP POST 登录、Cookie 磁盘持久化、注入工具） |

**`JtnAuthService` 能力：**

| 方法 | 说明 |
|------|------|
| `sso_login()` | 有头浏览器扫码 + 填账号密码，保存到磁盘 |
| `http_login()` | HTTP POST 登录（GET 登录页 → 提取 CSRF → POST 凭证） |
| `ensure_cookies()` | 优先缓存 → 过期则 SSO 登录 |
| `load_cookies()` / `save_cookies()` | 磁盘 Cookie 持久化 + 过期过滤 |
| `inject_to_httpx()` / `inject_to_context()` | Cookie 注入 httpx / Playwright |

### 案件导入（case_import）全部改 async

将 `jtn/case_import/` 模块（5 个文件 ~1700 行）从 sync 全部改为 async。（`657f5ad`）

| 文件 | 主要改动 |
|------|---------|
| `sso_handler.py` | `create_browser` → `create_browser_async`，`httpx.Client` → `AsyncClient` |
| `http_client.py` | `httpx.AsyncClient`，`ThreadPoolExecutor` → `asyncio.gather()` |
| `playwright_browser.py` | 全部 Playwright 调用加 `await`，12 个 `time.sleep` → `asyncio.sleep` |
| `detail_extractor.py` | 全部 Playwright DOM 查询加 `await` |
| `service.py` | 公开方法改 `async def`，`Generator` → `AsyncGenerator`，接入 `JtnAuthService` |

### 客户导入（client_import）全部改 async

将 `jtn/client_import/service.py`（~800 行）从 sync 改为 async。（`657f5ad`）

- `httpx.Client` → `httpx.AsyncClient`
- `create_browser()` → `create_browser_async()`
- `ThreadPoolExecutor` → `asyncio.gather()`
- `Generator` → `AsyncGenerator`
- 接入 `JtnAuthService.http_login()`

### 调用方适配

- `client_import_service.py`：`asyncio.run()` 桥接 async generator
- `case_import_service.py`：`asyncio.run()` 桥接 async generator
- `script_executor_service.py`：移除 `nest_asyncio`，保留 `asyncio.run()` + `DJANGO_ALLOW_ASYNC_UNSAFE`

## 技术要点

| 改动前 | 改动后 |
|--------|--------|
| Playwright **Sync API** | Playwright **Async API** |
| `nest_asyncio.apply()` | `asyncio.run()`（干净的事件循环隔离） |
| `ThreadPoolExecutor` 并发 | `asyncio.gather()` 并发 |
| 三套独立登录实现 | 统一 `JtnAuthService` |
| 无 Cookie 磁盘缓存（case_import/client_import） | 共享 `~/.fachuan/jtn_cookies.json` |
