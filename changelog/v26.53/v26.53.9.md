# v26.53.9 — 异步化改造 Batch 1

**日期**：2026-06-25
**类型**：性能优化
**分支**：perf/async-upgrade-batch-1

## 概述

针对 12 个后端 app 的第一轮异步化改造，消除 async 视图/函数中的同步阻塞操作，提升 ASGI 事件循环吞吐量。

## 改动详情

### Commit 1: cases + core — 原生 async ORM 替换 sync_to_async

- `cases/api/folder_binding_api.py`: CloudStorageAccount 查询从 `sync_to_async(filter.first)()` 改为原生 `await .afirst()`，消除不必要的线程池跳转
- `core/cloud_storage/browse_helper.py`: `abrowse_cloud_folder()` 中同样的模式替换

**影响**：减少每次请求的线程池调度开销

### Commit 2: automation — 关键函数异步化

- `automation/services/scraper/hbfy_scraper.py`: `_asave_page_state` 文件写入改为 `aiofiles`
- `automation/services/scraper/jysd_scraper.py`: `_asave_page_state` 文件写入改为 `aiofiles`
- `automation/services/chat/_wechat_work_file_mixin.py`: `_aupload_temp_material` 文件读取改为 `aiofiles`

**影响**：爬虫和飞书文件上传不再阻塞事件循环

### Commit 3: docspace + parsing + recognition + social_auth + oa_filing — P0+P1 改造

- **docspace/config.py**: 新增 async 变体配置函数（`aget_portal_url`, `aget_root_folder_id` 等）
- **docspace/api/docspace_api.py**: `_get_client()` 改为 `_aget_client()`，所有端点改用 async config
- **document_parsing/api/parsing_api.py**: `_save_upload` 调用改为 `sync_to_async` 包装
- **document_recognition/api/document_recognition_api.py**: `_save_uploaded_file` 调用改为 `sync_to_async` 包装
- **social_auth/views.py**: `exchange_code` → `aexchange_code`, `get_profile` → `aget_profile`
- **oa_filing/script_executor_service.py**: `_run_jtn` 全面改为 async ORM（`afilter`, `afirst`, `avalues_list`）
- **organization/account_credential_admin_service.py**: 补充 `asyncio.run()` 模式文档说明
- **legal_solution/tasks.py**: 补充 `time.sleep` 注释说明（Celery task 中保持同步）

**影响**：docspace 文件管理、文档解析/识别、社交登录、OA 立案等高频路径不再阻塞事件循环

## 技术模式

| 改造类型 | 方法 |
|---------|------|
| sync ORM → async ORM | `.filter().first()` → `await .afirst()` |
| sync_to_async → native async ORM | `sync_to_async(Model.objects.get)()` → `await Model.objects.aget()` |
| sync 文件 I/O → aiofiles | `open(path).read()` → `await aiofiles.open(path).read()` |
| sync config → async config | 新增 `aget_xxx()` async 变体函数 |
