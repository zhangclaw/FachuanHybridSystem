## 修复

### async_capable=True 导致 admin SynchronousOnlyOperation

`async_capable = True` + `markcoroutinefunction` 让 Django 把中间件当作原生 async 中间件，sync ORM 在事件循环线程执行触发 `SynchronousOnlyOperation`。移除 `async_capable`，让 Django 用 `sync_to_async` 包装。

### TruncDate 时区问题

`TruncDate("created_at")` 未指定 `tzinfo`，凌晨记录归到前一天。添加 `tzinfo=ZoneInfo(settings.TIME_ZONE)`。（`court_document_admin_service.py` + `performance_monitor.py`）

### 资源泄漏修复

| 问题 | 修复 |
|------|------|
| `OneDriveProvider` eager 创建 `AsyncClient` 从未关闭 | 改为 `_get_async_client()` 延迟创建 |
| `aembed_texts` 每次调用泄漏 `AsyncClient` | 添加 `try/finally + await client.close()` |
| WebDAV `_RateLimiter` 并发竞态 | 添加 `asyncio.Lock` |
| 聊聊 Token 刷新惊群效应 | DingTalk/WeChatWork/Feishu 三个 mixin 添加 `asyncio.Lock` + 双重检查 |

### 爬虫 async 审查修复
