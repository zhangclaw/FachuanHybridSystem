## [26.48.3] - 2026-05-11

### 前端

#### 新功能

- **消息来源支持创建和编辑**：新增表单对话框，创建模式含来源类型、关联凭证、轮询间隔等全部字段；编辑模式支持修改显示名称、轮询间隔、启用状态、同步起始时间、IMAP 配置、发件人白名单/黑名单
- **收件箱附件重命名**：附件卡片内嵌重命名输入框，扩展名锁定不可编辑，保存后自动刷新数据
- **消息来源同步按钮增加 toast 提示**：单个同步和全部同步操作均有成功/失败反馈

#### 优化

- **文件模板编辑页重构**：移除模板详情页，列表直接进入编辑页；表单改为 CaseForm 紧凑布局，去掉 CardHeader 和多余空白
- **模板库文件选择改为动态 API 获取**：移除硬编码选项，从后端 API 获取实际模板库文件列表；编辑模板时正确识别文件来源为"从模板库选择"
- **消息来源表单补充全部字段**：同步起始时间、IMAP 主机/账号、发件人白名单/黑名单、同步状态只读区
- **收件箱详情页重新设计**：对齐后端 admin 三段式布局（基本信息、正文、附件），去掉冗余的来源 Badge 重复和不可用的"快速操作"卡片
- **日志页图标按钮添加 tooltip 悬停提示**：时钟、删除、附件、提醒图标均有悬停文字说明

### 后端

#### 新功能

- **收件箱附件重命名 API**：新增 `POST /messages/{id}/attachments/{part_index}/rename` 端点，支持设置自定义文件名和恢复原名
- **模板库文件列表 API**：新增 `GET /templates/library-files` 端点，列出模板库中可用的 docx 文件

#### 修复

- **一张网附件下载前刷新 wjlj 链接**：OSS 预签名 URL 会过期，`download_attachment()` 本地文件缺失时先从法院 API 获取新的下载链接再下载，防止附件永久丢失

#### 优化

- **消息来源 API schema 扩展**：`MessageSourceOut` 新增 sync_since、imap_host、imap_account、sender_whitelist、sender_blacklist、last_sync_error 字段；`MessageSourceCreateIn` 和 `MessageSourceUpdateIn` 新增对应可写字段
