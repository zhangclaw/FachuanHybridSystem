# v26.51.4 - 2026-06-04

## 后端

### 新功能

- **S3 兼容存储接入**：新增 `S3Provider`，基于 boto3，支持 AWS S3 / Cloudflare R2 / MinIO / Backblaze B2 / 阿里云 OSS / 腾讯 COS 等所有 S3 兼容服务
  - `CloudStorageAccount` 新增 S3 字段：Access Key ID、Secret Access Key（加密）、Bucket 名称、Endpoint URL、Region、根路径
  - 通过 `endpoint_url` 参数切换不同 S3 服务，一个 Provider 覆盖所有兼容存储
- **Google Drive 接入**：新增 `GDriveProvider`，基于 Google Drive API v3 + Service Account 认证
  - `GDrivePathResolver` 将文件系统路径解析为 Google Drive 文件 ID，内置缓存避免重复查询
  - `walk` 加入 `visited set` 防止共享文件夹场景无限递归
  - Admin 上传 Service Account JSON 凭证（加密存储），目标文件夹共享给 SA 邮箱即可使用
- **Dropbox 接入**：新增 `DropboxProvider`，基于 Dropbox Python SDK + OAuth2 Device Code 授权
  - `DropboxOAuthTokenManager` 实现 device code flow（与 OneDrive 一致的 Admin UX）
  - Admin 后台一键获取设备码 → 用户浏览器授权 → 后台自动轮询保存 token
  - Access Token 自动刷新（4 小时有效期，Refresh Token 长期有效）
- **CloudStorageAccount 模型扩展**：新增 15 个字段（S3 6个 + GDrive 3个 + Dropbox 6个）+ 5 个加解密方法
- **StorageType 枚举扩展**：新增 `s3` / `google_drive` / `dropbox` 三个存储类型

### 优化

- **WebDAV 通用化**：将"坚果云 WebDAV"统一显示为"WebDAV"，不再绑定特定服务商
  - Admin fieldset 标题改为"WebDAV 设置"，配置指南同时展示坚果云和 123云盘
  - 前端下拉选项、Alpine.js widget、folder_browser.js 标签统一为"WebDAV"
  - 任意标准 WebDAV 服务（坚果云、123云盘、Nextcloud、NAS 等）只需填 URL + 凭证即可接入
- **WebDAV 错误信息通用化**：503 错误从"坚果云服务暂时不可用"改为"云存储服务暂时不可用"

### 修复

- **代码审查 CRITICAL/HIGH 修复**：
  - Dropbox `delete_file`：`is_path()` → `is_path_lookup()`（`DeleteError` 无 `is_path()` 方法，之前会 AttributeError 崩溃）
  - Dropbox SCOPE：补充 `files.metadata.read`（`exists/is_dir/get_file_info` 依赖此权限）
  - Dropbox `complete_device_code_flow`：接受 `interval` 参数，不再硬编码 5s sleep
  - Google Drive `_escape_gql`：补充反斜杠和双引号转义（之前有注入风险）
  - Google Drive factory：空 SA JSON 时返回 NullProvider（之前空 dict 传给 SA 会 KeyError 崩溃）
  - Google Drive `mkdir` 缓存 key：改为逐段递增格式（与 resolver 查询格式一致）
  - Google Drive `invalidate("/")`：不再清空 root 映射
  - S3 `exists()`：添加目录回退检测（`list_objects_v2` prefix 检查），修复 `is_dir()→exists()` 契约
  - S3 `get_file_info()`：仅捕获 404/NoSuchKey，不再吞掉 403 等权限错误
- **法院省份字段修复**：移除环境变量门控，始终启用一张网立案功能；补全全国 31 省市省份代码，修复非广东法院立案假死
- **CloudStorageAccount admin fieldset 标题**：从"坚果云 WebDAV"改为"WebDAV 设置"
- **Admin 配置指南**：新增 S3 / Google Drive / Dropbox 配置指南（含详细步骤和示例）
- **Admin JS toggle**：支持 S3 / Google Drive / Dropbox fieldset 和 help section 切换
- **Dropbox 授权区域**：Admin change form 新增 Dropbox device code auth UI（与 OneDrive 一致）
- **Admin list_display**：新增 Dropbox 状态列
- **绑定后重新选择文件夹打开本地浏览器修复**：4 个前端组件（React `CaseFolderSection`/`FolderBindingManager` + Alpine.js `folder_browser`/`case_folder_browser`）绑定加载后未同步 `storage_type` 和 `storage_account_id`，导致重新选择文件夹时始终以 local 模式打开本地文件系统浏览器；修复后从 binding 正确恢复存储类型和账号
- **绑定响应 Schema 补充 `storage_account_id`**：`FolderBindingResponseSchema` 和 `CaseFolderBindingResponseSchema` 的 `from_binding` 未返回 `storage_account_id`，导致前端无法获取已绑定的云存储账号 ID，点击"更换"时误报"请先选择云存储账号"
- **`list_active_cloud_accounts` 补全 root_path**：`browse_helper.py` 的 `.values()` 缺少 `s3_root_path`、`gdrive_root_path`、`dropbox_root_path`，导致新存储类型的云账号列表缺少根路径信息
