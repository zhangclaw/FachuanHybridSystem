# v26.51.20 (2026-06-12)

## Refactor: 验证码识别统一抽象层 + 手动输入模式

将分散在各爬虫中的 ddddocr 直接调用统一为 CaptchaRecognizer 抽象层，并新增手动验证码输入模式，支持用户在浏览器中人工填写验证码。

### 背景

此前 `hbfy_scraper.py` 直接 `import ddddocr` 绕过了 `CaptchaRecognizer` 抽象，且所有爬虫一律自动识别验证码，无法在识别率低时由人工介入。

### 变更

#### CaptchaRecognizer 抽象统一

- **hbfy_scraper.py**：移除直接 `import ddddocr`，改为通过 `self.captcha_recognizer.recognize()` 调用
- **4 个调用方**（`court_zxfw.py`、`court_zxfw_login_private/login_service.py`、`captcha_recognition_service.py`、`hbfy_scraper.py`）统一使用 `get_captcha_recognizer()` 工厂函数获取识别器

#### 手动验证码输入模式（ManualCaptchaRecognizer）

新增 `ManualCaptchaRecognizer`，当 `CAPTCHA_AUTO_RECOGNIZE` 配置为 `false` 时启用：

1. 爬虫调用 `recognize(image_bytes)` 时，将验证码图片保存到 `MEDIA_ROOT/automation/captcha_pending/`
2. ScraperTask 状态切换为 `WAITING_FOR_CAPTCHA`
3. 轮询等待用户通过 API 提交答案（默认超时 600s，轮询间隔 2s）
4. 用户提交后状态恢复为 `RUNNING`，爬虫继续执行

#### 工厂函数

```python
def get_captcha_recognizer(task=None) -> CaptchaRecognizer:
    # CAPTCHA_AUTO_RECOGNIZE=true  → DdddocrRecognizer（自动）
    # CAPTCHA_AUTO_RECOGNIZE=false → ManualCaptchaRecognizer（手动）
    # 无 task 时 fallback → DdddocrRecognizer
```

#### 数据库迁移

- `automation/migrations/0015_add_captcha_manual_fields.py`：
  - `ScraperTask.status` 新增 `waiting_for_captcha` 枚举值
  - `ScraperTask.captcha_image_path` — 待识别验证码图片路径
  - `ScraperTask.captcha_answer` — 用户提交的验证码答案

#### 手动验证码 API

新增 `captcha_manual_router`（前缀 `/captcha/manual`）：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/{task_id}/image` | 获取待识别验证码图片（FileResponse） |
| POST | `/{task_id}/answer` | 提交验证码答案，恢复任务运行 |

#### 监控服务

- `monitor_service.py`：统计字段新增 `waiting_for_captcha` 计数
- `WAITING_FOR_CAPTCHA` 状态不计入卡住检测（等待人工输入属正常状态）

## Feat: CAPTCHA_AUTO_RECOGNIZE 全局配置

在系统配置「爬虫配置」分组中新增 `CAPTCHA_AUTO_RECOGNIZE` 开关：

- 默认值：`false`（手动输入模式）
- 设为 `true` 后启用 ddddocr 自动识别
- 环境变量：`CAPTCHA_AUTO_RECOGNIZE`

`PLAYWRIGHT_HEADED` 初始化默认值从 `"false"` 改为 `"true"`（有头模式），方便调试爬虫时观察浏览器操作。

## Feat: 身份证合并支持跳过圈点

身份证合并页面（`id-card-merge`）圈定区域步骤改为可选：

- **之前**：必须点击 4 个角点才能合并，否则按钮禁用
- **之后**：圈点为可选操作，不圈点直接使用旋转后的完整图片合并

适用场景：用户只需旋转身份证方向，无需裁剪边框时，可直接点击「合并生成 PDF」。

### 变更细节

- 合并按钮 `disabled` 条件从 `frontCorners.length < 4 || backCorners.length < 4` 改为 `loading`
- `getCroppedImageData()` 在 `corners.length === 0` 时返回旋转后的完整图片
- 提示文案更新："如需裁剪，点击四个角；不圈点则使用旋转后的完整图片"
