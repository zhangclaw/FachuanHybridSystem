# v26.51.12 - 2026-06-08

## 后端

### 新功能
- **企查查 MCP Provider 接入**：完整实现企查查（QCC）企业数据服务，通过标准 MCP 协议对接 6 个 Server、181 个工具
  - `QichachaMcpProvider`：为每个 Server（company/risk/ipr/operation/executive/history）创建独立 `McpToolClient` 实例，通过路由表映射统一能力到对应工具
  - `QichachaResponseAdapter`：适配 QCC 中文字段名（`企业信息`、`统一社会信用代码`等）和嵌套响应格式
  - 路由映射：`search_companies` → `get_company_by_query`，`get_company_profile` → `get_company_registration_info` + `get_contact_info`（补充电话），`get_company_risks` → 按 risk_type 选择 7 种风险工具，`get_company_shareholders` → `get_shareholder_info`，`get_company_personnel` → `get_key_personnel`，`search_bidding_info` → `get_bidding_info`，`get_person_profile` → `get_executive_positions`
  - 多 key 自动切换：由 `McpApiKeyPool` 原生支持，SystemConfig 中每行一个 key
  - `QCC_MCP_API_KEY` 默认值故意写错末位字符，防止他人消耗额度
  - `QCC_MCP_BASE_URL` 固定为 `https://agent.qcc.com`，无需配置

### 优化
- **Provider Registry 双 provider 支持**：`EnterpriseProviderRegistry` 同时支持天眼查和企查查，`list_providers()` 动态检测 QCC API Key 是否已配置
  - `provider_registry.py`：新增 `QichachaMcpProvider` 导入和实例化逻辑
  - `get_provider(provider="qichacha")` 返回 `QichachaMcpProvider` 实例

### 新增
- **SystemConfig 默认配置**：`_service_configs.py` 新增 `QCC_MCP_API_KEY`（is_secret=True）
- **环境变量映射**：`_env_mappings.py` 新增 `QCC_MCP_API_KEY` 映射
- **Admin 表单多 key 支持**：`_MULTI_KEY_CONFIGS` 添加 `QCC_MCP_API_KEY`
