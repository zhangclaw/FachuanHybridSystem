# v26.51.18

## 新增

### 文档解析模块 (document_parsing)
- 新增 `apps.document_parsing` 模块，支持通过 MinerU 云端 API 解析文档
- 支持 PDF、DOC、DOCX、PPT、PPTX、XLS、XLSX、JPG、PNG 等格式
- MinerU 云端后端：通过 batch API 上传文件，MinerU 自动创建解析任务并轮询结果
- 本地 fallback 后端：PyMuPDF 提取 PDF 文本 + RapidOCR 图片识别
- Django Admin 工作台：上传文件、查看解析结果（纯文本/Markdown/元数据标签页）
- Ninja API 端点：`POST /parse`（文档解析）、`POST /extract-text`（文本提取）
- SystemConfig 集成：`MINERU_API_KEY`、`DOCUMENT_PARSING_BACKEND`（mineru/local/auto）
- 解析任务记录模型 `DocumentParsingTask`，存储状态、文本、Markdown、元数据

### 种子数据自动加载
- 案由（causes of action）、法院（courts）、LPR 利率首次 migrate 后自动导入
- 新增 `load_seed_data` / `export_seed_data` 管理命令
- 案由数据 4000+ 条、法院数据 3000+ 条、LPR 利率数据完整历史

## 修复

- MinerU `data_id` 超长报错：改用 `uuid4` 代替原始文件名（MinerU 限制 128 字符）
- MinerU 文件上传流程重写：使用 `/file-urls/batch` API，上传后 MinerU 自动创建任务，通过 `/extract-results/batch/{batch_id}` 轮询结果
- 解析详情页标签按钮添加 `type="button"` 防止触发表单 POST 导致 403

## 测试

- 新增 95 个单元测试覆盖 document_parsing 模块所有组件
- 覆盖：exceptions、protocols、schemas、models、parser_factory、document_parser、mineru_backend、local_backend
