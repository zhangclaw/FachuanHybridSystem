# v26.52.5 (2026-06-14)

## 移除 ddddocr 开源依赖，改用 plugins 子模块隔离

出于法律合规风险考量，将 ddddocr 验证码识别从开源仓库彻底移除，迁至私有 plugins 子模块（FachuanPlugins），开源版本自动降级为手动模式。

---

## 一、架构变更

| 改动 | 说明 |
|------|------|
| `DdddocrRecognizer` 迁移至 `plugins/captcha_ocr/` | 私有子模块 `Lawyer-ray/FachuanPlugins`，开源仓库零 ddddocr 引用 |
| `has_captcha_ocr_plugin()` 插件检测 | 子模块 `plugins/__init__.py` 中新增，复用已有 `has_xxx_plugin()` 模式 |
| `get_captcha_recognizer()` 重写 | 先检测插件 → 有则用 DdddocrRecognizer；无插件有 task → ManualCaptchaRecognizer；无插件无 task → raise |
| `CaptchaRecognitionService.recognizer` | 改为从插件动态加载，无插件时 raise RuntimeError |

---

## 二、移除项

| 移除 | 说明 |
|------|------|
| `ddddocr==1.6.1` 依赖 | `pyproject.toml` 和 `uv.lock` 均已清理 |
| `CAPTCHA_AUTO_RECOGNIZE` 系统配置 | 删除配置项及对应环境变量映射，插件存在即自动启用 |
| `DdddocrRecognizer` 类 | 主仓库 `captcha_recognizer.py` 中完整移除 |
| `core/__init__.py` 导出 | `__all__` 不再包含 `DdddocrRecognizer` |

---

## 三、降级行为

开源用户无插件时的预期行为：

| 场景 | 行为 |
|------|------|
| 有 task 参数 | 返回 `ManualCaptchaRecognizer`，等待用户手动输入验证码 |
| 无 task 参数 | 抛出 `RuntimeError`，提示安装插件或使用手动模式 |
| plugins 模块不存在 | 静默降级，不报错 |
